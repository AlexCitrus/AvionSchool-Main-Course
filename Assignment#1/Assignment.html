Don’t mix CSS or
Javascript with HTML !!!

As a markup language, the right use of HTML is to organize documents
by defining a header, a footer, lists, blockquotes, etc. The style
attribute allows developers to insert CSS directly into a html document
which is very useful for testing or when you’re in a hurry. But the style
attribute is bad practice because you will type styles in one line instead
of having a separate file for styles. Just like mixing your html code with
css is bad practice, you shouldn’t use any Javascript in your html documents.
